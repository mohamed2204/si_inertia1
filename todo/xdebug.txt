Méthode recommandée (propre et sûre)
1️⃣ Désactiver Xdebug pour PHP-FPM
sudo phpdismod xdebug
sudo systemctl restart php8.4-fpm

2️⃣ Redémarrer Apache (obligatoire)
sudo systemctl restart apache2

3️⃣ Vérifier (WEB et CLI)
CLI
php -v


➡️ Xdebug ne doit plus apparaître

WEB

Crée un fichier :

<?php phpinfo();


Dans le navigateur :

❌ Xdebug ne doit plus être listé

🔁 Alternative (si tu veux garder Xdebug installé)
Désactiver sans supprimer le module

Édite :

/etc/php/8.4/fpm/conf.d/20-xdebug.ini


Mets :

xdebug.mode=off


Redémarre :

sudo systemctl restart php8.4-fpm apache2

🎯 Désactiver seulement pour Apache (laisser CLI actif)
# FPM
/etc/php/8.4/fpm/conf.d/20-xdebug.ini
xdebug.mode=off

# CLI
/etc/php/8.4/cli/conf.d/20-xdebug.ini
xdebug.mode=debug

🧪 Test rapide de perf
ab -n 100 -c 10 http://localhost/


➡️ Gain immédiat si Xdebug était actif
